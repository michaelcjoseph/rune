## Planning real-world completion

- [2026-07-16 · source: execution-profiles-planning-review] When a promised outcome needs operator action, distinguish automatable preparation from live acceptance: require a runbook task naming the operator, environment, ordered steps, verification, retained evidence, and rollback, then a separate manual-live gate that assumes those steps are complete. A written runbook, fixture, or configuration never substitutes for real-world proof.
- [2026-07-16 · source: execution-profiles-planning-review] Before approving a plan with manual work, resolve the product decisions only the operator can make: which products are release-blocking versus migrated later, who performs each live action, what evidence is sufficient, and whether the gate blocks release. Unknown platform capabilities belong to a fail-closed engineering probe, not an invented product decision.
