Vanilla HTML/JS frontend renders markdown + syntax-highlighted code, makes `[[wikilinks]]` clickable `obsidian://` URIs, streams Claude responses over WebSocket, and adds multi-line input + Cmd+Enter, up-arrow recall, and a model dropdown. Phase D adds approval buttons for review flows and live agent-run events streamed to the sidebar. Phase E adds a chat-driven mutation pipeline (`MutationDescriptor` + `MutationApplier` registry, `logs/mutations.jsonl` persistence, `mutation-event` bus channel) with a `/work --auto` runner that executes any project in `docs/projects/*` from a webview button — framing the webview as a self-update surface for future mutation kinds (project-spec edits, proposal approvals, agent-file edits, cron toggles). |
| [07-spaced-repetition](07-spaced-repetition/spec.md) | Planned | Daily 12:00 CT spaced-repetition quiz over Telegram on non-stale wiki concepts. `sr-question-generator` composes open-ended questions, `sr-grader` scores free-form answers on a four-grade rubric (again/hard/good/easy), SR state advances on a fixed-interval ladder (1d → 3d → 7d → 14d → 30d → 60d → 120d). The existing `/study` (syllabus tracking) is renamed `/syllabus`; `/study` becomes the SR entry point with `/study N` and `/study status`. Phase 1 ships an ad-hoc manual session against a hand-seeded pool; Phase 2 adds the daily cron and 30-min lapse semantics; Phase 3 introduces a `status: evergreen \| active \| stale` field on wiki concept frontmatter, with `wiki-compiler` proposing (not applying) status changes via `knowledge/status-proposals.json` for weekly approval. Phase 4 polish (lapse hotspot report, graduation rule, revive-stale flow) deferred until usage data justifies. |
